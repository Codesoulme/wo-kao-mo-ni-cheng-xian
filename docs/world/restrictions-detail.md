# 禁制详情

> L2 数据契约文档（2026-06-27）。描述游戏中各类禁制（门/困/传送/封/卫/障）的判定规则。

## 禁制类型（6 种）

| 类型 | 名称 | 描述 | 常见开启方式 |
|------|------|------|-------------|
| door | 禁门 | 阻断通行的大门 | 钥匙/口令 |
| trap | 禁制陷阱 | 触发后造成伤害 | 战斗/时机 |
| transport | 传送阵 | 远程传送装置 | 令牌/身份 |
| seal | 封印 | 封印物品/通道 | 时机/战斗 |
| ward | 结界 | 范围防护阵 | 身份/口令 |
| barrier | 屏障 | 单向屏障 | 钥匙/战斗 |

## 开启方式（6 种）

| 方式 | 描述 | 校验逻辑 |
|------|------|---------|
| token | 令牌（可消耗） | 持特定令牌 ID |
| password | 口令 | 字符串匹配 |
| identity | 身份 | 境界/宗门匹配 |
| key | 钥匙（不消耗） | 持钥匙 ID |
| timing | 时机 | 当前时间在允许窗口 |
| combat | 战斗 | 战力 ≥ combatPower |

## 禁制来源

- **古修士遗留**：难度高，奖励丰厚
- **宗门设置**：身份要求（需宗门弟子）
- **天然形成**：灵气凝聚，需战斗破除
- **剧情触发**：任务/事件附带

## 开启结果

- **成功**：进入下一区域/获得奖励/触发剧情
- **失败**：可继续尝试，难度不递增
- **战斗破禁**：触发战斗事件
- **退去**：保持原状，可稍后再来

## 与洞府联动

- 洞府可设禁制列表（`restrictions` 字段）
- 进入洞府前需通过所有 `requiredRestrictionsPassed`
- 详情见 AI-71: 禁制 + 洞府联动
