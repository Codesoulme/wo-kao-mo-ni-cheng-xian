本目录用于保存《我靠模拟成仙》的设计与技术文档。

## 关键文件

- `DESIGN.md`
  当前设计文档升级版。它不是基准原文，而是基于 v1.0 技术文档和当前代码实现整理出的“实现对齐版”。

- `baseline/technical-design-v1.0-2026-06-17.txt`
  用户此前提供的《修仙模拟器统一技术设计文档 v1.0》的纯文本基准版。

- `baseline/technical-design-v1.0-2026-06-17.docx`
  用户此前提供的原始 Word 技术文档。

- `baseline-backups/README.baseline.20260619-154219.md`
  升级前仓库 README 的快照，只用于回看旧入口页，不是设计文档基准。

- GAP_ANALYSIS.md
  v1.0 技术文档与当前代码实现的差距分析，以及后续系统化补强优先级。

## 后续维护原则

1. 重大设计升级以 `baseline/technical-design-v1.0-2026-06-17.*` 为主基准。
2. `DESIGN.md` 负责沉淀“当前代码已经实现/正在实现/计划实现”的可执行设计。
3. 如果 v1.0 技术文档与当前 Next.js 实现不一致，应在文档中明确标注“原设计目标”和“当前实现取舍”。
4. 不覆盖基准文档，只新增对齐版、差异表或后续版本。
